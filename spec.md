## 2. プロダクト概要

### 2.1 サービス名

Rust Dojo

### 2.2 サービスコンセプト

Rust Dojo は、プログラミング基礎から Rust 言語習得、実務的コーディング、AtCoder 競技プログラミングまでを一気通貫で学べる日本向けハンズオン学習プラットフォームである。

### 2.3 提供価値

Rust Dojo が提供する価値は以下の通り。
1. プログラミング未経験者でも、変数・型・メモリ・制御構文などの基礎概念から学習できる
2. Rust の所有権・借用・ライフタイム・列挙型・トレイトなど、他言語と大きく異なる中核概念を段階的に学べる
3. 読むだけでなく、実行、修正、提出、採点を通じて理解を定着させる
4. Cargo, test, fmt, clippy を含む実務的な開発習慣を身につけられる
5. AtCoder を見据えた Rust の書き方、テンプレート、典型問題、過去問演習に接続できる
6. 学習ログに基づき、弱点概念に戻る循環導線を持つ

### 2.4 プロダクトの差別化

Rust 公式は Learn ページで The Book、Rust by Example、Rustlings を案内しており、Rust 学習資源はすでに充実している一方、未経験者向けのプログラミング前提知識、Rust 実務導線、AtCoder Rust 導線を単一プロダクトで統合した日本向けサービスは薄い。Rust Dojo はそこを主戦場とする。

差別化軸は以下。
- 未経験者向けの前提知識教育
- Rust コンパイラエラーの教材化
- 実務と競プロの両立
- 日本語特化
- 学習ログに基づく復習誘導
- AtCoder の Rust 環境を前提にした実戦対応

## 3. プロダクト目標

### 3.1 事業目標
- 日本語圏で Rust 学習の第一想起サービスを目指す
- 未経験者〜中級者の継続学習率を高める
- 将来的に有料講座・実務トラック・模試・高機能分析を収益化する
- 日本語圏の Rust 学習コミュニティ形成の基盤となる

### 3.2 学習目標

利用者が以下の状態に到達すること。
- コンピュータとプログラムの基本概念を説明できる
- Rust の基本文法を理解し、自力でコードを書ける
- 所有権・借用・エラー処理を用いて安全な Rust を書ける
- 実務的な Rust の開発フローを理解し、保守可能なコードを書ける
- AtCoder の初中級問題を Rust で解ける
- エラーを読んで自己修正できる

### 3.3 成果指標
- 初回学習完了率
- 7日継続率
- 30日継続率
- レッスン完了率
- 問題再挑戦率
- AC 到達率
- 所有権・借用関連問題の正答率
- 実務トラック完了率
- AtCoder トラック完了率
- 有料転換率

### 4.4 ペルソナD: AtCoder 志向者

## 6. 基本方針

### 6.1 言語方針
- UI: 日本語
- レッスン本文: 日本語
- 問題文: 日本語
- コード: Rust
- 用語: 日本語主体、一部英語を併記
- エラーメッセージ: 原文表示 + 日本語解説

### 6.2 Rust バージョン方針
- 講義標準: Rust 1.90.0, Edition 2024 基準
- 競プロ標準: AtCoder 現行サポート環境 rustc 1.89.0 基準

### 6.3 学習設計方針
- 1回の学習単位を小さくする
- 読むだけで終わらせない
- すぐ実行させる
- エラーを避けるのでなく、読ませる
- 学んだ概念を問題で使わせる
- 問題で詰まったら概念に戻す
- “理解したつもり”を防ぐ

## 13. 実行・採点仕様

### 13.1 基本方針

Rust Dojo は未信頼コードを扱うため、コード実行は安全なサンドボックス下で行う。Judge0 はサンドボックス付きコンパイル・実行、時間/メモリ制限、詳細な実行結果、追加ファイル対応、多言語対応を備えたオンラインコード実行基盤であり、本プロダクトの実行基盤候補として適切である。

### 13.2 実行モード

A. 軽量実行
用途:
- 座学中の小さなコード確認
- 1 ファイル完結
- 簡易入力

仕様:
- ソースコード実行
- stdout/stderr 表示
- コンパイルエラー表示
- 実行時間短め
- レート制限あり

B. 提出採点
用途:
- 実装問題
- 競プロ問題
- 総合課題

仕様:
- 複数テストケース
- 時間制限
- メモリ制限
- コンパイル結果
- 実行結果
- AC / WA / TLE / MLE / RE / CE 表示
- ケースごとの詳細
- 最終スコア保存

### 13.3 実行基盤要件
- サンドボックス実行
- ネットワーク遮断
- ファイルアクセス制限
- プロセス制限
- タイムアウト
- メモリ上限制御
- 実行ログ取得
- 実行キュー処理
- 監査ログ保存

Judge0 公式は、サンドボックスコンパイル・実行、時間/メモリ制限、詳細な実行結果、追加ファイルサポートを挙げている。

### 13.4 結果表示

表示項目:
- ステータス
- 実行時間
- メモリ使用量
- 標準出力
- 標準エラー
- コンパイルメッセージ
- ケース別結果
- 失敗理由
- 関連講義リンク

### 13.5 コンパイルエラー教材化

CE 時には以下を表示する。
- 原文
- 重要箇所ハイライト
- 日本語補足
- 典型原因
- 修正例の方向性
- 関連レッスン

Rust 公式 Learn ページは Compiler Error Index を主要文書として案内しており、Rust Dojo でもエラー理解を学習の中心に置く。

- 長時間かかった問題

- 誤答しやすい理由も書く

### 33.3 Rust 特化ルール
- 値の所有者を説明する
- 参照の生存範囲を可視化する
- コンパイラエラーを隠さない
- 借用規則を図解する

## 36. リスクと対策

### 36.1 リスク
- 未経験者には Rust が難しい
- 所有権で離脱しやすい
- 採点基盤コスト
- 競プロと実務の両立で内容が散漫化
- 日本語教材の整備負荷

### 36.2 対策
- Track 0 を厚くする
- 所有権を段階分割する
- CE を教材化する
- 実務と競プロを別トラックに明確化する
- MVP で範囲を絞る

## 38. 最終定義

Rust Dojo は、単なる Rust 教材サイトではなく、以下を満たすべきである。
1. 未経験者にも開かれている
2. Rust の難所を逃げずに扱う
3. 読むだけで終わらない
4. 実務と競プロの両方へ伸びる
5. 日本語で一貫して学べる
6. エラーを学習資産に変える
7. 学習ログに基づき再挑戦へ導く
